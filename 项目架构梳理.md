# Zero-Demo 项目架构梳理（2026-07-12 更新）

## 一、项目概述

本项目是基于 **go-zero** 框架构建的微服务电商系统，采用 API Gateway + RPC 服务的分层架构，包含用户、商品、订单、支付、通知等核心业务模块。

## 三、核心业务流程

### 3.1 用户注册/登录流程

```
客户端 → gateway-api → user.rpc → MySQL
    ↓              ↓           ↓
  注册/登录      验证参数      创建用户/验证密码
    ↓              ↓           ↓
返回用户信息   生成 JWT      返回用户数据
    ↓
返回 accessToken
```

### 3.2 创建订单流程（修改后）

```
客户端(带JWT) → gateway-api → order.rpc → product.rpc
    │                │             │              │
    │                │             │       ① 查询商品信息
    │                │             │       ② 原子扣减库存(WHERE stock >= amount)
    │                │             │              │
    │                │             ↓              ↓
    │                │     创建订单记录           返回结果
    │                │             │
    │                │             ↓
    │                │     ③ 发送延迟消息(RabbitMQ, 15分钟)
    │                │             │
    ↓                ↓             ↓
返回订单ID    提取userId    返回订单信息
              从JWT
```

**关键变化**：
- ✅ 从 JWT Token 提取 userId 传递给订单服务
- ✅ 库存扣减使用原子 SQL 更新（`WHERE stock >= amount`）
- ✅ 创建订单后发送延迟消息用于超时取消
- ✅ 订单号使用 `ORD{timestamp}{4位序列号}` 格式

### 3.3 创建支付流程（修改后）

```
客户端(带JWT) → gateway-api → payment.rpc → order.rpc
    │                │             │              │
    │                │             │       ① 查询订单信息
    │                │             │       ② 验证订单状态(PENDING)
    │                │             │       ③ 验证金额一致性
    │                │             │       ④ 验证用户归属
    │                │             │              │
    │                │             ↓              ↓
    │                │     创建支付记录           返回订单信息
    │                │             │
    ↓                ↓             ↓
返回支付ID    提取userId    返回支付信息
              从JWT           OrderNo从订单获取
```

**关键变化**：
- ✅ 从 JWT Token 提取 userId
- ✅ 添加订单归属验证（防止越权支付）
- ✅ OrderNo 从订单信息获取，不再依赖前端传入

### 3.4 订单超时取消流程（新增）

```
订单创建 → RabbitMQ 延迟队列(15分钟) → OrderTimeoutConsumer
    │               │                           │
    │               ↓                           ↓
    │         消息过期                         查询订单
    │               │                           │
    │               ↓                           ↓
    │         死信交换机                   判断状态(PENDING?)
    │               │                           │
    │               ↓                   ┌───────┴───────┐
    │         超时队列                  是              否
    │               │                   ↓              ↓
    │               ↓              回滚库存          跳过取消
    │               │                   │
    └───────────────┼───────────────────↓
                    ↓              取消订单(状态改为CANCELLED)
               消费者消费
```

## 四、架构变化对比

### 4.1 修复前的问题

| 问题 | 影响 | 严重程度 |
|------|------|----------|
| 逻辑文件为空 | 请求无法处理，编译错误 | 致命 |
| 用户 ID 缺失 | 订单无法关联用户 | 致命 |
| 库存扣减竞态 | 高并发下超卖 | 致命 |
| RabbitMQ 未实现 | 订单无法超时取消 | 致命 |
| 限流内存泄漏 | 长期运行内存耗尽 | 中等 |
| 订单号重复 | 同一毫秒请求重复 | 中等 |
| 支付越权 | 用户可支付他人订单 | 中等 |

### 4.2 修改后的架构改进

| 改进点 | 修改内容 | 文件位置 |
|--------|----------|----------|
| **逻辑结构统一** | 补全所有空逻辑文件，删除死代码 | `app/gateway/api/internal/logic/*` |
| **用户身份传递** | 从 JWT 提取 userId 传递给 RPC | `createorderlogic.go`, `createpaymentlogic.go` |
| **库存原子扣减** | SQL 添加 `WHERE stock >= amount` | `product_model.go`, `deduct_stock_logic.go` |
| **延迟队列实现** | RabbitMQ 死信交换机 + TTL | `rabbitmq.go`, `order_timeout_consumer.go` |
| **限流内存管理** | 定期清理 + 最大条目限制(10000) | `ratelimitmiddleware.go` |
| **订单号去重** | 添加原子序列号 | `create_logic.go` |
| **支付权限验证** | 验证订单归属 | `payment/create_logic.go` |

## 八、服务启动顺序

```bash
# 1. 启动基础设施
cd deploy/docker && docker compose up -d

# 2. 启动 RPC 服务（顺序不严格，但建议先启动被依赖的服务）
go run app/user/rpc/user.go -f app/user/rpc/etc/user.yaml &
go run app/product/rpc/product.go -f app/product/rpc/etc/product.yaml &
go run app/order/rpc/order.go -f app/order/rpc/etc/order.yaml &
go run app/payment/rpc/payment.go -f app/payment/rpc/etc/payment.yaml &
go run app/notification/rpc/notification.go -f app/notification/rpc/etc/notification.yaml &

# 3. 启动网关
go run app/gateway/api/gateway.go -f app/gateway/api/etc/gateway-api.yaml
```
