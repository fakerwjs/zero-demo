# Zero-Demo 项目执行文档

## 1. 系统架构说明

### 1.1 架构总览

本项目采用微服务架构，基于 Go-zero 框架构建。主要包含以下核心组件：

| 层级 | 组件 | 技术栈 | 职责 |
|------|------|--------|------|
| API 网关层 | gateway-api | Go + Go-zero | 请求路由、认证鉴权、限流熔断 |
| 业务服务层 | user-api/user-rpc, product-rpc, order-rpc, payment-rpc, notification-rpc | Go + Go-zero + gRPC | 业务逻辑实现 |
| 注册中心 | Etcd | Etcd | 服务发现、配置存储 |
| 消息队列 | RabbitMQ | RabbitMQ | 异步消息、延迟队列 |
| 缓存层 | Redis | Redis | 数据缓存、会话管理 |
| 存储层 | MySQL, MongoDB, MinIO | MySQL, MongoDB, MinIO | 持久化存储 |
| 监控层 | Prometheus, Grafana, Jaeger | Prometheus, Grafana, Jaeger | 监控告警、链路追踪 |
| 日志层 | ELK (Filebeat + Elasticsearch + Kibana) | ELK Stack | 日志采集、存储、分析 |

### 1.2 服务间调用关系

```
gateway-api (:8000)
    │
    ├── user-rpc (:8081)        # 用户服务
    ├── notification-rpc (:8005) # 通知服务
    ├── product-rpc (:8006)     # 商品服务
    ├── order-rpc (:8007)       # 订单服务
    │       └── product-rpc     # 库存扣减
    └── payment-rpc (:8008)     # 支付服务
            └── order-rpc       # 更新订单状态

notification-rpc → RabbitMQ → 异步通知投递
```

```go
func (l *RegisterLogic) Register(in *user.RegisterReq) (*user.RegisterResp, error) {
    // 1. 检查用户名是否已存在
    _, err := l.svcCtx.UserModel.FindOneByUsername(l.ctx, in.Username)
    if err == nil {
        return nil, status.Error(codes.AlreadyExists, "用户名已存在")
    }
    if err != model.ErrNotFound {
        return nil, status.Error(codes.Internal, err.Error())
    }

    // 2. 检查手机号是否已存在
    _, err = l.svcCtx.UserModel.FindOneByMobile(l.ctx, in.Mobile)
    if err == nil {
        return nil, status.Error(codes.AlreadyExists, "手机号已被注册")
    }

    // 3. 密码加密
    encryptedPassword, err := bcrypt.GenerateFromPassword([]byte(in.Password), bcrypt.DefaultCost)
    if err != nil {
        return nil, status.Error(codes.Internal, "密码加密失败")
    }

    // 4. 插入用户数据
    data := &model.User{
        Username: in.Username,
        Password: string(encryptedPassword),
        Mobile:   in.Mobile,
    }
    _, err = l.svcCtx.UserModel.Insert(l.ctx, data)
    if err != nil {
        return nil, status.Error(codes.Internal, err.Error())
    }

    return &user.RegisterResp{
        Id:       data.Id,
        Username: data.Username,
        Mobile:   data.Mobile,
    }, nil
}
```

```go
    }, nil
}
```

```go
func (m *customProductModel) DeductStock(ctx context.Context, id int64, quantity int64) error {
    _, err := m.ExecCtx(ctx, func(ctx context.Context, conn sqlx.SqlConn) (result sqlx.Result, err error) {
        // 乐观锁扣减库存
        query := `UPDATE product SET stock = stock - ?, update_at = ? WHERE id = ? AND stock >= ?`
        result, err = conn.ExecCtx(ctx, query, quantity, time.Now().Unix(), id, quantity)
        return
    })
    if err != nil {
        return err
    }
    return nil
}
```

```go
func (l *CreateOrderLogic) Create(in *order.CreateOrderReq) (*order.CreateOrderResp, error) {
    // 1. 获取当前用户 ID（从上下文获取）
    userId := int64(1)

    // 2. 遍历订单项，扣减库存并计算总价
    totalPrice := int64(0)
    for _, item := range in.Items {
        // 调用商品服务扣减库存
        _, err := l.svcCtx.ProductRpc.DeductStock(l.ctx, &product.DeductStockReq{
            Id:       item.ProductId,
            Quantity: item.Quantity,
        })
        if err != nil {
            return nil, status.Error(codes.Internal, "库存扣减失败: "+err.Error())
        }
        totalPrice += item.Price * item.Quantity
    }

    // 3. 生成订单号
    orderNo := generateOrderNo()

    // 4. 插入订单数据
    data := &model.Order{
        UserId:     userId,
        OrderNo:    orderNo,
        TotalPrice: totalPrice,
        Status:     int64(order.OrderStatus_ORDER_STATUS_PENDING),
        Items:      marshalItems(in.Items),
    }
    _, err := l.svcCtx.OrderModel.Insert(l.ctx, data)
    if err != nil {
        return nil, status.Error(codes.Internal, err.Error())
    }

    return &order.CreateOrderResp{
        Id:      data.Id,
        OrderNo: data.OrderNo,
        Success: true,
    }, nil
}
```

```go
func (l *NotifyLogic) Notify(in *payment.NotifyPaymentReq) (*payment.NotifyPaymentResp, error) {
    // 1. 查询支付记录
    item, err := l.svcCtx.PaymentModel.FindOne(l.ctx, in.Id)
    if err != nil {
        return nil, status.Error(codes.NotFound, "支付记录不存在")
    }

    // 2. 如果已成功，直接返回
    if item.Status == int64(payment.PaymentStatus_PAYMENT_STATUS_SUCCESS) {
        return &payment.NotifyPaymentResp{Success: true}, nil
    }

    // 3. 更新支付状态
    now := time.Now().Unix()
    if err := l.svcCtx.PaymentModel.UpdateStatus(l.ctx, in.Id, int64(in.Status), in.TransactionId, now); err != nil {
        return nil, status.Error(codes.Internal, err.Error())
    }

    // 4. 如果支付成功，更新订单状态
    if in.Status == payment.PaymentStatus_PAYMENT_STATUS_SUCCESS {
        _, err := l.svcCtx.OrderRpc.UpdateStatus(l.ctx, &order.UpdateOrderStatusReq{
            Id:     item.OrderId,
            Status: order.OrderStatus_ORDER_STATUS_PAID,
        })
        if err != nil {
            l.Errorf("update order status failed: %v", err)
        }
    }

    return &payment.NotifyPaymentResp{Success: true}, nil
}
```

**通知发送逻辑** (`app/notification/rpc/internal/logic/send_logic.go`):

```go
func (l *SendLogic) Send(in *notification.SendReq) (*notification.SendResp, error) {
    // 1. 获取当前用户 ID
    userId := int64(1)

    // 2. 保存通知记录到 MySQL
    data := &model.Notification{
        UserId:   userId,
        Title:    in.Title,
        Content:  in.Content,
        Channel:  int64(in.Channel),
        IsRead:   false,
    }
    _, err := l.svcCtx.NotificationModel.Insert(l.ctx, data)
    if err != nil {
        return nil, status.Error(codes.Internal, err.Error())
    }

    // 3. 异步投递到 RabbitMQ
    msg, _ := json.Marshal(map[string]interface{}{
        "id":      data.Id,
        "userId":  userId,
        "title":   in.Title,
        "content": in.Content,
        "channel": in.Channel,
    })

    // 使用延迟队列（如果需要）
    if err := l.svcCtx.Publisher.PublishWithDelay(l.ctx, msg, 0); err != nil {
        l.Errorf("publish notification failed: %v", err)
    }

    // 4. 记录审计日志到 MongoDB
    logData := map[string]interface{}{
        "notificationId": data.Id,
        "userId":         userId,
        "channel":        in.Channel,
        "status":         "queued",
        "createAt":       time.Now().Unix(),
    }
    l.svcCtx.MongoClient.InsertOne(l.ctx, logData)

    return &notification.SendResp{
        Id:      data.Id,
        Success: true,
    }, nil
}
```

**RabbitMQ 延迟队列实现** (`pkg/mq/rabbitmq.go`):

```go
func (p *Publisher) PublishWithDelay(ctx context.Context, body []byte, delaySec int64) error {
    p.mu.Lock()
    defer p.mu.Unlock()

    if err := p.ensure(); err != nil {
        return err
    }

    publishing := amqp.Publishing{
        ContentType:  "application/json",
        DeliveryMode: amqp.Persistent,
        Timestamp:    time.Now(),
        Body:         body,
    }

    // 如果设置了延迟时间，发送到延迟队列
    if delaySec > 0 && p.delayQueue != "" && p.delayExchange != "" {
        publishing.Expiration = strconv.FormatInt(delaySec*1000, 10)
        return p.ch.PublishWithContext(ctx, p.delayExchange, p.delayQueue, false, false, publishing)
    }

    // 否则发送到普通队列
    return p.ch.PublishWithContext(ctx, "", p.queue, false, false, publishing)
}
```

## 4. 核心技术实现

### 4.1 JWT 认证

**实现位置**: `pkg/jwtx/jwtx.go`

```go
func GenerateToken(secret string, iat, expire int64) (string, error) {
    claims := jwt.MapClaims{
        "exp": expire,
        "iat": iat,
    }
    token := jwt.NewWithClaims(jwt.SigningMethodHS256, claims)
    return token.SignedString([]byte(secret))
}

func ParseToken(secret, tokenString string) (jwt.MapClaims, error) {
    token, err := jwt.Parse(tokenString, func(token *jwt.Token) (interface{}, error) {
        if _, ok := token.Method.(*jwt.SigningMethodHMAC); !ok {
            return nil, fmt.Errorf("unexpected signing method: %v", token.Header["alg"])
        }
        return []byte(secret), nil
    })
    if err != nil {
        return nil, err
    }
    if claims, ok := token.Claims.(jwt.MapClaims); ok && token.Valid {
        return claims, nil
    }
    return nil, fmt.Errorf("invalid token")
}
```

**使用方式**:
- API 网关配置 JWT 中间件
- 路由标记 `@server (jwt: Auth)` 表示需要鉴权
- 请求头携带 `Authorization: Bearer <token>`

### 4.2 限流熔断

```go
func NewRateLimitMiddleware(rate, burst int) *RateLimitMiddleware {
    return &RateLimitMiddleware{
        limiter: rate.NewLimiter(rate.Limit(rate), burst),
    }
}

func (m *RateLimitMiddleware) Handle(next http.HandlerFunc) http.HandlerFunc {
    return func(w http.ResponseWriter, r *http.Request) {
        if !m.limiter.Allow() {
            httpx.Error(w, status.Error(codes.ResourceExhausted, "请求过于频繁"))
            return
        }
        next(w, r)
    }
}
```

**配置方式**:

## 9. 扩展开发指南

### 9.1 添加新服务

1. **创建 RPC 服务**:
   ```bash
   mkdir -p app/new-service/rpc
   cd app/new-service/rpc
   goctl rpc protoc new-service.proto --go_out=. --go-grpc_out=. --zrpc_out=.
   ```

2. **添加数据库表**:
   - 在 `deploy/mysql/init/` 目录下添加 SQL 文件
   - 使用 goctl 生成 model

3. **配置服务**:
   - 创建 `etc/new-service.yaml` 配置文件
   - 在 `deploy/prometheus/prometheus.yml` 添加监控目标
   - 在 `scripts/run-all.sh` 和 `scripts/stop-all.sh` 添加服务启动/停止

4. **集成到网关**:
   - 在 `gateway.api` 添加 API 定义
   - 在 `gateway-api.yaml` 添加 RPC 配置
   - 在 `config.go` 和 `service_context.go` 添加服务引用
   - 创建 handler 和 logic 文件

### 9.2 添加新接口

1. **修改 API 定义**:
   ```yaml
   @server (
       group: user
       prefix: /api/v1/user
       jwt: Auth
   )
   service gateway-api {
       @handler NewHandler
       post /new (NewReq) returns (NewResp)
   }
   ```

2. **重新生成代码**:
   ```bash
   goctl api go -api gateway.api -dir . --style gozero
   ```

3. **实现业务逻辑**:
   - 修改 `internal/logic/user/new_logic.go`
   - 调用后端 RPC 服务

### 9.3 添加新中间件

1. **创建中间件文件**:
   ```bash
   touch app/gateway/api/internal/middleware/newmiddleware.go
   ```

2. **实现中间件**:
   ```go
   type NewMiddleware struct{}

   func NewNewMiddleware() *NewMiddleware {
       return &NewMiddleware{}
   }

   func (m *NewMiddleware) Handle(next http.HandlerFunc) http.HandlerFunc {
       return func(w http.ResponseWriter, r *http.Request) {
           // 中间件逻辑
           next(w, r)
       }
   }
   ```

3. **注册中间件**:
   - 在 `internal/svc/service_context.go` 中注册
   - 在路由配置中使用

## 10. 性能优化建议

### 10.1 数据库优化

- **索引优化**: 为经常查询的字段添加索引
- **读写分离**: 配置 MySQL 主从复制，读操作路由到从库
- **连接池**: 合理配置数据库连接池大小

### 10.2 缓存优化

- **热点数据缓存**: 使用 Redis 缓存高频访问数据
- **缓存策略**: 合理设置缓存过期时间，避免缓存击穿/雪崩
- **缓存预热**: 启动时预加载热点数据

### 10.3 服务优化

- **限流降级**: 配置合理的限流策略，避免服务过载
- **异步处理**: 将非核心业务异步化（如通知、日志）
- **负载均衡**: 使用 Etcd 服务发现实现负载均衡

### 10.4 监控优化

- **告警规则**: 配置关键指标的告警规则
- **日志采样**: 生产环境降低日志采样率
- **指标聚合**: 合理聚合监控指标，减少存储开销

## 11. 安全注意事项

### 11.1 认证安全

- **JWT 密钥**: 生产环境使用强密钥，定期轮换
- **Token 过期**: 设置合理的 Token 过期时间
- **HTTPS**: 生产环境必须使用 HTTPS

### 11.2 数据安全

- **密码加密**: 使用 bcrypt 加密存储密码
- **敏感数据**: 避免日志中打印敏感信息
- **SQL 注入**: 使用参数化查询，避免 SQL 注入

### 11.3 网络安全

- **端口限制**: 生产环境只开放必要端口
- **防火墙**: 配置防火墙规则，限制访问来源
- **内网隔离**: 微服务之间使用内网通信

### 11.4 依赖安全

- **依赖扫描**: 定期扫描第三方依赖漏洞
- **版本更新**: 及时更新依赖版本
- **最小权限**: 服务运行使用最小权限用户
