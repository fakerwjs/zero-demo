## 一、本次新增/变更了什么

### 1. 安装 Docker Desktop

- 通过 `winget install --id Docker.DockerDesktop` 安装 **Docker Desktop 4.81.0**（引擎 29.6.1，WSL2/Ubuntu 后端），并启动 daemon。

### 2. 全套基础设施（docker-compose）

`deploy/docker/docker-compose.yml` 一键拉起：

| 组件 | 镜像 | 端口 | 说明 |
|------|------|------|------|
| etcd | quay.io/coreos/etcd:v3.5.17 | 2379 | 服务注册发现 |
| MySQL | mysql:8.0 | 3306 | 主存储，库 `zero_demo`，启动即执行 `deploy/mysql/init/*.sql` 建表 |
| Redis | redis:7 | 6379 | go-zero model 缓存 |
| RabbitMQ | rabbitmq:3.13-management | 5672 / 15672 | 通知异步队列 / 管理台 |
| MongoDB | mongo:7 | 27017 | 预留文档存储 |
| MinIO | minio/minio | 9000 / 9001 | 预留对象存储 |
| Jaeger | jaegertracing/all-in-one:1.57 | 16686 / 4317 | 链路追踪 |
| Prometheus | prom/prometheus:v2.53.0 | 9090 | 指标采集（抓 9101-9103） |
| Grafana | grafana/grafana:11.1.0 | 3000 | 监控大盘，预置 Prometheus 数据源 |

配套配置：`deploy/prometheus/prometheus.yml`、`deploy/grafana/provisioning/datasources/datasource.yml`、`deploy/mysql/init/01_user.sql`、`02_notification.sql`。

### 3. 数据层（goctl model + 缓存）

- `goctl model mysql ddl -c` 从 DDL 离线生成带 **Redis 缓存** 的 model：
  - `app/user/rpc/internal/model/`（user 表，含 `FindOneByUsername/FindOneByMobile`）
  - `app/notification/rpc/internal/model/`（notification 表）
- 通知 model 手动扩展了 `ListByUserId` / `CountByUserId`（分页查询）。

### 4. 用户服务业务落地

- `user.rpc`：
  - `Register`：用户名唯一性校验 + **bcrypt 密码哈希** + 落库
  - `Login`：查库 + bcrypt 校验密码（令牌仍由 api 层统一签发 JWT）
  - `GetUserInfo`：按 id 查库
  - 接入 MySQL（`DataSource`）+ Redis（`Cache`）
- `user-api`：新增 Telemetry（Jaeger）+ Prometheus 配置。

### 5. 通知服务业务落地

- `notification.rpc`：
  - `Send`：落库 + **发布消息到 RabbitMQ**（异步投递）
  - `GetNotification` / `ListByUser`：真实查库、分页
  - 启动时拉起 **RabbitMQ 消费者**（`pkg/mq`），模拟按渠道真正投递
- 新增 `pkg/mq/rabbitmq.go`：惰性连接的 Publisher + 自动重连的 Consumer。

### 6. 可观测性

- 三个服务均配置 `Telemetry`（OTLP gRPC → Jaeger `:4317`）与 `Prometheus`（`:9101/9102/9103`）。
- Prometheus 通过 `host.docker.internal` 抓取宿主机上服务指标。

### 7. 依赖新增

`golang.org/x/crypto`（bcrypt）、`github.com/rabbitmq/amqp091-go`、`github.com/go-sql-driver/mysql`。

## 四、如何运行

```bash
# 1. 起基础设施
cd deploy/docker && docker compose up -d && cd ../..
# 2. 起服务
bash scripts/run-all.sh
# 3. 停止
bash scripts/stop-all.sh
cd deploy/docker && docker compose down   # -v 清数据
```

- Jaeger UI  http://localhost:16686
- Prometheus http://localhost:9090
- Grafana    http://localhost:3000 (admin/admin)
- RabbitMQ   http://localhost:15672 (guest/guest)
- MinIO      http://localhost:9001 (minioadmin/minioadmin)

### 可观测性

- Jaeger：已收到 3 个服务的链路 `["notification.rpc","user.rpc","user-api"]` ✅

> 备注：从 git-bash 用 curl 访问 Docker 发布的 Web 端口（9090/16686 等）会返回 `000`，这是 git-bash 下 curl 与 Docker Desktop 端口代理的兼容问题，**用浏览器访问正常**；Go 服务连 MySQL/Redis/etcd 等发布端口均正常。

### 网络踩坑（重要）
- 本机 Docker Hub 被墙（`registry-1.docker.io` DNS 被污染成不响应的 IPv6），大镜像拉不下来。
- 解决：给 Docker Desktop 配置镜像加速源 `https://docker.m.daocloud.io`（写入 `~/.docker/daemon.json` 的 `registry-mirrors`，**停 Docker→改文件→启 Docker** 才不会被覆盖），随后 `docker compose up -d` 顺利拉全。
- etcd 用的是 `quay.io/coreos/etcd`（非 Docker Hub），所以最早就能拉到。
