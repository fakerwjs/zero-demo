# Zero-Demo 项目架构梳理

## 1. 项目概述

Zero-Demo 是基于 **go-zero** 框架构建的微服务电商系统，采用分层架构设计，支持高并发、高可用的生产环境部署。

### 1.1 技术栈

| 分类 | 技术 | 版本 | 说明 |
|------|------|------|------|
| 框架 | go-zero | v1.10.2 | 微服务框架 |
| 语言 | Go | 1.25+ | 编程语言 |
| RPC | gRPC + Protocol Buffers | - | 服务间通信 |
| 网关 | APISIX + go-zero Gateway | - | 统一入口、路由转发 |
| 服务发现 | etcd | v3.5.17 | 服务注册发现 |
| 数据库 | MySQL | 8.0 | 主存储 |
| 缓存 | Redis | 7 | 数据缓存 |
| 消息队列 | RabbitMQ | 3.13 | 异步消息 |
| 对象存储 | MinIO | latest | 文件存储 |
| 文档数据库 | MongoDB | 7 | 文档存储 |
| 链路追踪 | Jaeger | 1.57 | 全链路监控 |
| 监控 | Prometheus | v2.53.0 | 指标采集 |
| 可视化 | Grafana | 11.1.0 | 监控大盘 |
| 日志 | ELK Stack | 8.13.0 | 日志管理 |

### 1.2 核心特性

- **分层架构**：API 网关层 + RPC 业务层 + 基础设施层
- **服务发现**：基于 etcd 的自动服务注册与发现
- **链路追踪**：全链路追踪，支持分布式问题排查
- **监控告警**：完善的指标采集和告警体系
- **日志管理**：统一日志采集、存储和分析
- **异步处理**：基于 RabbitMQ 的异步消息处理
- **负载均衡**：APISIX + Nginx 双层负载均衡

## 3. 核心模块设计

### 3.1 API 网关层

**gateway-api** 是系统的统一 API 入口，负责：

| 功能 | 实现方式 | 说明 |
|------|----------|------|
| 路由转发 | go-zero HTTP Handler | 根据路径路由到不同 RPC 服务 |
| JWT 认证 | go-zero Middleware | 验证用户 Token |
| 请求日志 | go-zero Logger | 记录请求信息 |
| 链路追踪 | Jaeger SDK | 生成 Trace ID |
| 协议转换 | go-zero zrpc | HTTP 请求转 gRPC 调用 |

**配置示例** (`app/gateway/api/etc/gateway-api.yaml`):

```yaml
Name: gateway-api
ListenOn: ${GATEWAY_PORT:0.0.0.0:8000}

Prometheus:
  Host: ${PROMETHEUS_HOST:0.0.0.0}
  Port: ${GATEWAY_PROM_PORT:9100}
  Path: /metrics

Telemetry:
  Name: gateway-api
  Endpoint: ${JAEGER_ENDPOINT:127.0.0.1:4317}
  Sampler: ${JAEGER_SAMPLER:1.0}
  Batcher: otlpgrpc

UserRpc:
  Etcd:
    Hosts:
      - ${ETCD_HOST:127.0.0.1}:${ETCD_PORT:2379}
    Key: user.rpc

ProductRpc:
  Etcd:
    Hosts:
      - ${ETCD_HOST:127.0.0.1}:${ETCD_PORT:2379}
    Key: product.rpc
```

### 3.2 用户服务

**user.rpc** 提供用户核心业务逻辑：

| 功能 | RPC 接口 | 说明 |
|------|----------|------|
| 用户注册 | Register | 创建新用户，密码加密 |
| 用户登录 | Login | 验证密码，返回用户信息 |
| 获取用户信息 | GetUserInfo | 根据 ID 查询用户 |
| 更新用户信息 | UpdateUser | 更新用户资料 |

**数据模型** (`app/user/rpc/internal/model/user_model.go`):

```go
type User struct {
    Id        int64     `db:"id"`
    Username  string    `db:"username"`
    Password  string    `db:"password"`
    Mobile    string    `db:"mobile"`
    CreatedAt int64     `db:"created_at"`
    UpdatedAt int64     `db:"updated_at"`
}
```

### 3.3 商品服务

**product.rpc** 提供商品管理和库存管理：

| 功能 | RPC 接口 | 说明 |
|------|----------|------|
| 创建商品 | CreateProduct | 创建商品信息 |
| 查询商品 | GetProduct | 根据 ID 查询商品 |
| 商品列表 | ListProduct | 分页查询商品列表 |
| 更新商品 | UpdateProduct | 更新商品信息 |
| 删除商品 | DeleteProduct | 删除商品 |
| 扣减库存 | DeductStock | 乐观锁扣减库存 |

**库存扣减机制**：

```go
func (m *customProductModel) DeductStock(ctx context.Context, id int64, quantity int64) error {
    _, err := m.ExecCtx(ctx, func(ctx context.Context, conn sqlx.SqlConn) (result sqlx.Result, err error) {
        query := `UPDATE product SET stock = stock - ?, update_at = ? WHERE id = ? AND stock >= ?`
        result, err = conn.ExecCtx(ctx, query, quantity, time.Now().Unix(), id, quantity)
        return
    })
    return err
}
```

### 3.4 订单服务

**order.rpc** 提供订单管理：

| 功能 | RPC 接口 | 说明 |
|------|----------|------|
| 创建订单 | CreateOrder | 创建订单，扣减库存 |
| 查询订单 | GetOrder | 根据 ID 查询订单 |
| 订单列表 | ListOrder | 分页查询用户订单 |
| 取消订单 | CancelOrder | 取消未支付订单 |
| 更新订单状态 | UpdateOrderStatus | 更新订单状态 |

**订单状态流转**：

```
待支付 (ORDER_STATUS_PENDING)
    │
    ├── 支付成功 → 已支付 (ORDER_STATUS_PAID)
    │
    └── 取消订单 → 已取消 (ORDER_STATUS_CANCELLED)

已支付 (ORDER_STATUS_PAID)
    │
    ├── 发货 → 已发货 (ORDER_STATUS_SHIPPED)
    │
    └── 退款 → 已退款 (ORDER_STATUS_REFUNDED)

已发货 (ORDER_STATUS_SHIPPED)
    │
    └── 确认收货 → 已完成 (ORDER_STATUS_COMPLETED)
```

### 3.5 支付服务

**payment.rpc** 提供支付管理：

| 功能 | RPC 接口 | 说明 |
|------|----------|------|
| 创建支付 | CreatePayment | 创建支付记录 |
| 支付回调 | Notify | 处理支付通知 |
| 查询支付 | GetPayment | 查询支付状态 |

### 3.6 通知服务

**notification.rpc** 提供异步通知功能：

| 功能 | RPC 接口 | 说明 |
|------|----------|------|
| 发送通知 | Send | 发送通知，入队 RabbitMQ |
| 获取通知 | GetNotification | 查询通知详情 |
| 通知列表 | ListByUser | 查询用户通知列表 |

**异步通知流程**：

```
notification-rpc.Send
    │
    ├── MySQL (保存通知记录)
    │
    └── RabbitMQ (发布消息)
            │
            └── 消费者 (投递通知: 短信/邮件/推送)
```
