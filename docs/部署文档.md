## 2. 环境准备

### 2.1 系统要求

| 组件 | 版本要求 | 说明 |
|------|---------|------|
| Go | 1.21+ | 编程语言 |
| Docker | 20.10+ | 容器化部署 |
| Docker Compose | 2.0+ | 编排工具 |
| Git | 2.30+ | 版本控制 |

### 2.2 工具安装

#### Windows 环境

```bash
# 安装 Go（下载地址：https://go.dev/dl/）
# 安装 Docker Desktop（下载地址：https://www.docker.com/products/docker-desktop/）
# 安装 Git（下载地址：https://git-scm.com/downloads）

# 验证安装
go version
docker --version
docker compose version
git --version
```

#### Linux/macOS 环境

```bash
# Ubuntu/Debian
sudo apt update && sudo apt install -y golang docker.io docker-compose-plugin git

# macOS (Homebrew)
brew install go docker git
```

### 2.3 克隆项目

```bash
git clone https://github.com/fakerwjs/zero-demo.git
cd zero-demo
```

## 4. 服务部署

### 4.1 一键启动所有服务

```bash
bash scripts/run-all.sh
```

脚本会自动完成：
1. 检查/启动 Etcd
2. 编译所有服务
3. 启动所有 RPC 服务（user-rpc, notification-rpc, product-rpc, order-rpc, payment-rpc）
4. 启动 API 服务（user-api, gateway-api）

### 4.2 手动启动（可选）

#### 4.2.1 编译服务

```bash
go build -o .run/bin/user-rpc.exe         ./app/user/rpc
go build -o .run/bin/notification-rpc.exe ./app/notification/rpc
go build -o .run/bin/product-rpc.exe      ./app/product/rpc
go build -o .run/bin/order-rpc.exe        ./app/order/rpc
go build -o .run/bin/payment-rpc.exe      ./app/payment/rpc
go build -o .run/bin/user-api.exe         ./app/user/api
go build -o .run/bin/gateway-api.exe       ./app/gateway/api
```

#### 4.2.2 启动 RPC 服务

```bash
# user-rpc (:8081)
cd app/user/rpc && .run/bin/user-rpc.exe -f etc/user.yaml

# notification-rpc (:8005)
cd app/notification/rpc && .run/bin/notification-rpc.exe -f etc/notification.yaml

# product-rpc (:8006)
cd app/product/rpc && .run/bin/product-rpc.exe -f etc/product.yaml

# order-rpc (:8007)
cd app/order/rpc && .run/bin/order-rpc.exe -f etc/order.yaml

# payment-rpc (:8008)
cd app/payment/rpc && .run/bin/payment-rpc.exe -f etc/payment.yaml
```

#### 4.2.3 启动 API 服务

```bash
# user-api (:8001)
cd app/user/api && .run/bin/user-api.exe -f etc/user-api.yaml

# gateway-api (:8000)
cd app/gateway/api && .run/bin/gateway-api.exe -f etc/gateway-api.yaml
```

### 4.3 停止服务

```bash
bash scripts/stop-all.sh
```

### 4.4 验证服务状态

```bash
# 检查服务进程
netstat -ano | findstr "8000 8001 8005 8006 8007 8008 8081"

# 验证 API 服务
curl http://localhost:8000/api/v1/product/list
curl http://localhost:8001/api/v1/user/login

# 验证 Prometheus 指标
curl http://localhost:9100/metrics  # gateway-api
curl http://localhost:9101/metrics  # user-api
curl http://localhost:9102/metrics  # user-rpc
curl http://localhost:9103/metrics  # notification-rpc
curl http://localhost:9104/metrics  # product-rpc
curl http://localhost:9105/metrics  # order-rpc
curl http://localhost:9106/metrics  # payment-rpc
```

## 5. 配置说明

### 5.1 环境变量配置

所有配置文件支持环境变量覆盖，格式为 `${ENV_VAR:default}`。

#### 数据库配置

| 环境变量 | 默认值 | 说明 |
|---------|--------|------|
| MYSQL_HOST | 127.0.0.1 | MySQL 主机 |
| MYSQL_PORT | 3306 | MySQL 端口 |
| MYSQL_USER | root | MySQL 用户 |
| MYSQL_PASSWORD | root123456 | MySQL 密码 |
| MYSQL_DATABASE | zero_demo | 数据库名 |

#### Redis 配置

| 环境变量 | 默认值 | 说明 |
|---------|--------|------|
| REDIS_HOST | 127.0.0.1 | Redis 主机 |
| REDIS_PORT | 6379 | Redis 端口 |

#### Etcd 配置

| 环境变量 | 默认值 | 说明 |
|---------|--------|------|
| ETCD_HOST | 127.0.0.1 | Etcd 主机 |

## 10. 常见问题

### 10.1 端口冲突

如果端口被占用，可以修改配置文件中的端口或停止占用进程：

```bash
# 查找占用端口的进程
netstat -ano | findstr ":8000"

# 杀死进程（Windows）
taskkill /PID <进程ID> /F
```

### 10.2 服务启动失败

查看日志文件：

```bash
cat .run/gateway-api.log
cat .run/user-rpc.log
```

### 10.3 Etcd 连接失败

确保 Etcd 服务已启动：

```bash
docker compose up -d etcd
```

### 10.4 数据库连接失败

检查 MySQL 服务状态和配置：

```bash
docker compose ps mysql
```

### 10.5 Prometheus 无法抓取指标

确保服务已启动且 Prometheus 配置中的目标地址正确。
