# Zero-Demo 快速启动指南

## 1. 环境准备

### 1.1 必备工具

| 工具 | 版本要求 | 说明 |
|------|---------|------|
| Go | 1.25+ | 编程语言 |
| Docker | 20.10+ | 容器化部署 |
| Docker Compose | 2.0+ | 编排工具 |
| Git | 2.30+ | 版本控制 |

### 1.2 工具安装

#### Windows 环境

1. 下载并安装 [Go](https://go.dev/dl/)
2. 下载并安装 [Docker Desktop](https://www.docker.com/products/docker-desktop/)
3. 下载并安装 [Git](https://git-scm.com/downloads)

#### Linux/macOS 环境

```bash
# Ubuntu/Debian
sudo apt update && sudo apt install -y golang docker.io docker-compose-plugin git

# macOS (Homebrew)
brew install go docker git
```

### 1.3 验证安装

```bash
go version          # 输出: go version go1.25+ ...
docker --version    # 输出: Docker version 20.10+ ...
docker compose version  # 输出: Docker Compose version v2.0+ ...
git --version       # 输出: git version 2.30+ ...
```

### 1.4 克隆项目

```bash
git clone https://github.com/fakerwjs/zero-demo.git
cd zero-demo
```

## 2. 启动基础设施

### 2.1 一键启动

```bash
cd deploy/docker
docker compose up -d
```

### 2.2 验证基础设施状态

```bash
# 查看所有容器状态
docker compose ps

# 验证关键服务
curl http://localhost:2379/version           # Etcd
curl http://localhost:3306                   # MySQL
redis-cli ping                               # Redis
curl http://localhost:5672                   # RabbitMQ
```

## 3. 启动应用服务

### 3.1 一键启动（推荐）

```bash
bash scripts/run-all.sh
```

脚本会自动完成：
1. 检查/启动 Etcd
2. 编译所有服务
3. 启动所有 RPC 服务（user-rpc, notification-rpc, product-rpc, order-rpc, payment-rpc）
4. 启动 API 服务（user-api, gateway-api）

### 3.2 验证服务状态

```bash
# 检查服务进程
netstat -ano | findstr "8000 8001 8005 8006 8007 8008 8081"

# 验证 API 服务
curl http://localhost:8000/api/v1/product/list
curl http://localhost:8001/api/v1/user/login

# 验证 Prometheus 指标
curl http://localhost:9100/metrics  # gateway-api
curl http://localhost:9101/metrics  # user-api
curl http://localhost:9102/metrics  # user-rpc
```

### 3.3 服务列表

| 服务 | 类型 | 端口 | 指标端口 | 说明 |
|------|------|------|----------|------|
| gateway-api | HTTP | 8000 | 9100 | API 网关 |
| user-api | HTTP | 8001 | 9101 | 用户 API |
| user.rpc | gRPC | 8081 | 9102 | 用户 RPC |
| notification.rpc | gRPC | 8005 | 9103 | 通知 RPC |
| product.rpc | gRPC | 8006 | 9104 | 商品 RPC |
| order.rpc | gRPC | 8007 | 9105 | 订单 RPC |
| payment.rpc | gRPC | 8008 | 9106 | 支付 RPC |

## 4. API 接口测试

### 4.1 用户注册

```bash
curl -X POST http://localhost:8000/api/v1/user/register \
  -H 'Content-Type: application/json' \
  -d '{"username":"alice","password":"123456","mobile":"13800000000"}'
```

**响应示例**:
```json
{
  "id": 1,
  "username": "alice",
  "mobile": "13800000000"
}
```

### 4.2 用户登录

```bash
curl -X POST http://localhost:8000/api/v1/user/login \
  -H 'Content-Type: application/json' \
  -d '{"username":"alice","password":"123456"}'
```

**响应示例**:
```json
{
  "id": 1,
  "accessToken": "eyJhbGciOiJIUzI1NiIs...",
  "accessExpire": 1718000000
}
```

### 4.3 获取用户信息

```bash
curl http://localhost:8000/api/v1/user/info \
  -H 'Authorization: Bearer <accessToken>'
```

**响应示例**:
```json
{
  "id": 1,
  "username": "alice",
  "mobile": "13800000000"
}
```

### 4.4 商品列表

```bash
curl http://localhost:8000/api/v1/product/list
```

**响应示例**:
```json
{
  "list": [],
  "total": 0
}
```

### 4.5 创建商品（需鉴权）

```bash
curl -X POST http://localhost:8000/api/v1/product/create \
  -H 'Content-Type: application/json' \
  -H 'Authorization: Bearer <accessToken>' \
  -d '{"name":"iPhone 15","price":5999,"stock":100}'
```

**响应示例**:
```json
{
  "id": 1,
  "name": "iPhone 15",
  "price": 5999,
  "stock": 100
}
```

### 4.6 创建订单（需鉴权）

```bash
curl -X POST http://localhost:8000/api/v1/order/create \
  -H 'Content-Type: application/json' \
  -H 'Authorization: Bearer <accessToken>' \
  -d '{"items":[{"productId":1,"quantity":1,"price":5999}]}'
```

**响应示例**:
```json
{
  "id": 1,
  "orderNo": "202401011200000001",
  "success": true
}
```

## 5. 管理界面

### 5.1 服务管理界面

| 服务 | 地址 | 用户名/密码 |
|------|------|-------------|
| etcdkeeper | http://localhost:8080 | - |
| Grafana | http://localhost:3000 | admin/admin |
| Jaeger | http://localhost:16686 | - |
| RabbitMQ | http://localhost:15672 | guest/guest |
| MinIO | http://localhost:9001 | minioadmin/minioadmin |
| Kibana | http://localhost:5601 | - |

### 5.2 查看服务注册状态

通过 **etcdkeeper** 查看服务注册：

1. 打开 http://localhost:8080
2. 连接 etcd：输入 `http://localhost:2379`
3. 查看已注册的服务键值

### 5.3 查看监控指标

通过 **Prometheus** 查看指标：

1. 打开 http://localhost:9090
2. 在查询框输入 `up` 查看服务状态
3. 在查询框输入 `http_requests_total` 查看请求总数

### 5.4 查看链路追踪

通过 **Jaeger** 查看追踪数据：

1. 打开 http://localhost:16686
2. 选择服务名（如 gateway-api, user-rpc）
3. 点击 "Find Traces" 查看追踪数据

## 7. 常见问题

### 7.1 端口冲突

**问题**: 端口被占用

**解决方案**:
```bash
# 查找占用端口的进程（Windows）
netstat -ano | findstr ":8000"

# 杀死进程（Windows）
taskkill /PID <进程ID> /F

# Linux/macOS
lsof -i :8000
kill -9 <进程ID>
```

### 7.2 Docker 启动失败

**问题**: Docker 容器启动失败

**解决方案**:
```bash
# 查看容器日志
docker logs <容器名>

# 常见问题：内存不足
# 解决方案：增加 Docker 内存限制（Docker Desktop 设置）
```

### 7.3 服务无法连接 etcd

**问题**: 服务启动后无法注册到 etcd

**解决方案**:
```bash
# 检查 etcd 是否正常运行
curl http://localhost:2379/health

# 检查服务配置中的 etcd 地址
cat app/user/rpc/etc/user.yaml | grep -A5 Etcd
```

### 7.4 数据库连接失败

**问题**: 服务无法连接 MySQL

**解决方案**:
```bash
# 检查 MySQL 容器状态
docker compose ps mysql

# 检查 MySQL 日志
docker logs zero-mysql

# 验证 MySQL 连接
mysql -h localhost -u root -proot123456
```

### 7.5 服务编译失败

**问题**: Go 编译报错

**解决方案**:
```bash
# 安装依赖
go mod tidy

# 检查 Go 版本
go version

# 检查代码语法
go vet ./...
```
