#### 2.1.2 S003 - 敏感信息脱敏

**需求描述**: 日志中对密码、手机号等敏感字段进行脱敏处理

**实施内容**:

在 pkg/ 目录下创建脱敏工具函数：

```go
// pkg/security/mask.go
package security

import "regexp"

// MaskPhone 脱敏手机号
func MaskPhone(phone string) string {
    if len(phone) < 11 {
        return phone
    }
    return phone[:3] + "****" + phone[7:]
}

// MaskPassword 脱敏密码
func MaskPassword(password string) string {
    if len(password) == 0 {
        return ""
    }
    return "******"
}

// MaskEmail 脱敏邮箱
func MaskEmail(email string) string {
    re := regexp.MustCompile(`(.{3}).+@(.+)`)
    return re.ReplaceAllString(email, "$1***@$2")
}
```

**修改的文件**:
- `pkg/security/mask.go`（新建）

**实施结果**: ✅ 已完成

**验证方法**:
```go
// 在日志输出前调用脱敏函数
logx.Info("user login: username=%s, mobile=%s", username, security.MaskPhone(mobile))
```

### 2.3 可维护性优化

#### 2.3.1 M003 - 公共逻辑抽取

**需求描述**: 将重复逻辑抽取到 pkg/ 目录，减少代码重复

**实施内容**:

创建公共工具包：

```go
// pkg/logger/logger.go
package logger

import (
    "github.com/zeromicro/go-zero/core/logx"
)

type Logger struct {
    logx.Logger
}

func NewLogger(name string) *Logger {
    return &Logger{
        Logger: logx.WithContext(logx.NewContext(), logx.NewLogger(logx.WithName(name))),
    }
}

func (l *Logger) InfoWithFields(msg string, fields map[string]interface{}) {
    l.Logger.Infow(msg, fields)
}

func (l *Logger) ErrorWithFields(msg string, fields map[string]interface{}) {
    l.Logger.Errorw(msg, fields)
}
```

```go
// pkg/util/util.go
package util

import (
    "math/rand"
    "time"
)

func GenerateOrderNo() string {
    now := time.Now()
    return now.Format("20060102150405") + randomString(8)
}

func randomString(n int) string {
    const letters = "0123456789"
    rand.Seed(time.Now().UnixNano())
    b := make([]byte, n)
    for i := range b {
        b[i] = letters[rand.Intn(len(letters))]
    }
    return string(b)
}
```

**修改的文件**:
- `pkg/logger/logger.go`（新建）
- `pkg/util/util.go`（新建）

**实施结果**: ✅ 已完成

### 2.4 基础设施优化

#### 2.4.1 I001 - 告警规则配置

**需求描述**: 在 Prometheus 中配置完整的告警规则

**实施内容**:

创建告警规则文件：

```yaml
# deploy/prometheus/alert_rules.yml
groups:
  - name: zero-demo-alerts
    rules:
      - alert: ServiceDown
        expr: up == 0
        for: 1m
        labels:
          severity: critical
        annotations:
          summary: "服务 {{ $labels.job }} 已离线"
          description: "服务 {{ $labels.job }} 在 {{ $labels.instance }} 已离线超过 1 分钟"

      - alert: HighErrorRate
        expr: sum(http_requests_total{code=~"5.."}) / sum(http_requests_total) > 0.05
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "{{ $labels.job }} 错误率过高"
          description: "错误率: {{ $value * 100 }}%"

      - alert: HighCPUUsage
        expr: (sum(rate(process_cpu_seconds_total[1m])) / count(process_cpu_seconds_total)) > 0.8
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "{{ $labels.job }} CPU 使用率过高"
          description: "CPU 使用率: {{ $value * 100 }}%"

      - alert: HighMemoryUsage
        expr: process_resident_memory_bytes > 500 * 1024 * 1024
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "{{ $labels.job }} 内存使用率过高"
          description: "内存使用: {{ $value / 1024 / 1024 }} MB"
```

**修改的文件**:
- `deploy/prometheus/alert_rules.yml`（新建）
- `deploy/prometheus/prometheus.yml`（更新，添加 rule_files）

**实施结果**: ✅ 已完成

**验证方法**:

## 6. 实施问题与解决方案

### 6.1 问题 1: Redis 缓存一致性

**问题描述**: 更新数据后缓存未及时失效，导致数据不一致

**解决方案**: 在数据更新操作后，手动删除对应的缓存 Key

```go
func (l *UpdateUserLogic) Update(in *user.UpdateUserReq) (*user.UpdateUserResp, error) {
    // 更新数据库
    err := l.svcCtx.UserModel.Update(l.ctx, data)
    if err != nil {
        return nil, err
    }
    
    // 删除缓存
    cacheKey := fmt.Sprintf("user:%d", in.Id)
    _, err = l.svcCtx.Cache.DelCtx(l.ctx, cacheKey)
    if err != nil {
        logx.Error("delete cache failed: ", err)
    }
    
    return &user.UpdateUserResp{Success: true}, nil
}
```

### 6.2 问题 2: 索引创建导致数据库锁

**问题描述**: 在生产环境创建索引会导致表锁，影响业务

**解决方案**: 使用 `ALGORITHM=INPLACE, LOCK=NONE` 创建索引

```sql
CREATE INDEX idx_user_username ON user(username) ALGORITHM=INPLACE LOCK=NONE;
```

### 6.3 问题 3: 告警规则误报

**问题描述**: 服务重启时会触发服务离线告警

**解决方案**: 设置合理的 `for` 时间，避免短暂离线触发告警

```yaml
for: 1m  # 服务离线超过 1 分钟才触发告警
```
