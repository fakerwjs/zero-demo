# zero-demo

基于 [go-zero](https://github.com/zeromicro/go-zero) 的微服务脚手架。当前只包含 **用户服务** 与 **通知服务** 两个服务，其余服务（订单、商品、支付、API 网关等）暂未创建。

## 目录结构

```
zero-demo/
├── app/
│   ├── user/
│   │   ├── api/            # 用户 HTTP API（对外，端口 8001）
│   │   └── rpc/            # 用户 gRPC（内部，端口 8081）
│   └── notification/
│       └── rpc/            # 通知 gRPC（内部，端口 8005）
├── pkg/                    # 预留：可对外复用的公共库
├── deploy/                 # 预留：docker / k8s 部署配置
├── docs/                   # 文档（含每次创建记录）
├── Makefile
├── go.mod
└── README.md
```

## 服务与端口

### 应用服务

| 服务             | 类型 | 端口 | 指标端口 | etcd Key         |
|------------------|------|------|----------|------------------|
| user-api         | HTTP | 8001 | 9101     | -                |
| user.rpc         | gRPC | 8081 | 9102     | user.rpc         |
| notification.rpc | gRPC | 8005 | 9103     | notification.rpc |

### 基础设施（docker-compose，见 `deploy/docker/`）

| 组件       | 端口               | 用途 / 访问                                   |
|------------|--------------------|-----------------------------------------------|
| etcd       | 2379               | 服务注册发现                                  |
| MySQL      | 3306               | 主存储（库 `zero_demo`，root/root123456）     |
| Redis      | 6379               | model 缓存                                     |
| RabbitMQ   | 5672 / 15672       | 通知异步队列 / 管理台 (guest/guest)           |
| MongoDB    | 27017              | 预留文档存储 (root/root123456)                |
| MinIO      | 9000 / 9001        | 预留对象存储 / 控制台 (minioadmin/minioadmin) |
| Jaeger     | 16686 / 4317       | 链路追踪 UI / OTLP 上报                        |
| Prometheus | 9090               | 指标采集（抓取 9101-9103）                     |
| Grafana    | 3000               | 监控大盘 (admin/admin)                         |

## 依赖

- Go 1.21+
- goctl 1.10+
- protoc + protoc-gen-go + protoc-gen-go-grpc
- Docker Desktop（拉起基础设施）

> 提示 1：goctl 生成代码时会调用 `protoc` 及插件，需保证它们在 PATH 中。
>
> 提示 2（国内网络）：Docker Hub 常被墙导致镜像拉不下来。给 Docker 配置镜像加速源后再拉：
> ```bash
> # 停 Docker → 写配置 → 启 Docker（顺序很重要，运行中改会被覆盖）
> docker desktop stop
> #  在 ~/.docker/daemon.json 中加入：
> #  "registry-mirrors": ["https://docker.m.daocloud.io"]
> docker desktop start
> docker info | grep -A1 "Registry Mirrors"   # 确认生效
> ```

## 快速开始

```bash
# 1. 启动全套基础设施（etcd/mysql/redis/rabbitmq/mongo/minio/jaeger/prometheus/grafana）
cd deploy/docker && docker compose up -d && cd ../..

# 2. 一键编译并启动三个服务（脚本内已处理 PATH，并自动复用 docker 里的 etcd）
bash scripts/run-all.sh

# 停止应用服务
bash scripts/stop-all.sh
# 停止基础设施（加 -v 清空数据卷）
cd deploy/docker && docker compose down
```

## 接口示例

```bash
# 注册
curl -X POST http://localhost:8001/api/v1/user/register \
  -H 'Content-Type: application/json' \
  -d '{"username":"alice","password":"123456","mobile":"13800000000"}'

# 登录（返回 accessToken）
curl -X POST http://localhost:8001/api/v1/user/login \
  -H 'Content-Type: application/json' \
  -d '{"username":"alice","password":"123456"}'

# 查询用户信息（需带 JWT）
curl http://localhost:8001/api/v1/user/info \
  -H 'Authorization: Bearer <accessToken>'
```
